# Система управления персоналом

Веб-приложение для учёта, анализа и визуализации данных о персонале предприятия. Построено на Django 6.0 с PostgreSQL.

## Назначение

Система предназначена для отделов кадров производственных предприятий и решает следующие задачи:

- Централизованное хранение данных о сотрудниках (приём, увольнение, перемещения)
- Массовый импорт данных из файлов Excel и CSV
- Формирование отчётов по численности и движению персонала
- Визуализация ключевых HR-метрик на интерактивном дашборде
- Экспорт отчётов в Excel
- Ролевая модель доступа (администратор, директор по персоналу, редактор, просмотр)

## Стек технологий

- **Backend:** Python 3.12+, Django 6.0
- **База данных:** PostgreSQL
- **Frontend:** HTML/CSS, Bootstrap 5.3, Bootstrap Icons, HTMX
- **Графики:** Chart.js 4.x (локальная копия, без CDN)
- **Обработка файлов:** pandas, openpyxl
- **Шрифты:** Inter, JetBrains Mono (локальные копии)

## Установка и запуск

### Предварительные требования

- Python 3.12 или новее
- PostgreSQL 14 или новее
- pip

### Шаг 1. Клонирование и виртуальное окружение

```bash
git clone <url-репозитория>
cd personnel_management

python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux / macOS
source .venv/bin/activate
```

### Шаг 2. Установка зависимостей

```bash
pip install django psycopg2-binary pandas openpyxl
```

### Шаг 3. Настройка базы данных

Создайте базу данных PostgreSQL:

```sql
CREATE DATABASE personnel_management;
```

Параметры подключения задаются в `config/settings.py` в секции `DATABASES`. По умолчанию:

- **Имя БД:** personnel_management
- **Пользователь:** postgres
- **Пароль:** postgres
- **Хост:** localhost
- **Порт:** 5432

### Шаг 4. Применение миграций

```bash
python manage.py migrate
```

Миграции автоматически создадут все таблицы и настроят группы пользователей (Администратор, Директор по персоналу, Редактор, Просмотр).

### Шаг 5. Создание суперпользователя

```bash
python manage.py createsuperuser
```

### Шаг 6. Запуск

```bash
python manage.py runserver
```

Приложение доступно по адресу `http://127.0.0.1:8000/`.

## Модели данных

### Справочники

- **Production** — производства (например, «Литейное производство»)
- **Workshop** — цеха, привязаны к производству (номер + название)
- **Position** — должности
- **EmployeeCategory** — категории работников (РСС, Рабочие)
- **DismissalReason** — причины увольнения

### Основные модели

- **Employee** — сотрудник (ФИО, табельный номер, даты приёма/увольнения, связи со справочниками)
- **UserProfile** — профиль пользователя системы (ФИО, привязка к User)

## Загрузка данных

Система поддерживает массовый импорт сотрудников из файлов CSV, XLSX и XLS через страницу «Загрузка файлов».

### Обязательные колонки

- ФИО
- Табельный номер
- Дата приема на работу

### Необязательные колонки

- Дата рождения
- Дата увольнения
- Производство
- Цех
- Причина увольнения
- Категория рабочего
- Должность

При загрузке файла система автоматически создаёт отсутствующие записи в справочниках (производства, цеха, должности и т.д.). Если сотрудник с таким табельным номером уже существует, его данные обновляются.

## Роли и доступ

Доступ к функциям определяется группой пользователя в Django:

| Функция                                      | Администратор | Директор по персоналу | Редактор | Просмотр |
| --------------------------------------------------- | -------------------------- | ---------------------------------------- | ---------------- | ---------------- |
| Загрузка файлов                       | Да                       | Да                                     | Да             | Нет           |
| Просмотр сотрудников             | Да                       | Да                                     | Да             | Да             |
| Редактирование сотрудников | Да                       | Да                                     | Да             | Нет           |
| Дашборд и отчёты                      | Да                       | Да                                     | Да             | Да             |
| Экспорт в Excel                             | Да                       | Да                                     | Да             | Да             |
| Админ-панель Django                      | Да (is_staff)            | Нет                                   | Нет           | Нет           |

Группы и их права создаются автоматически при миграции (`0003_setup_groups`).

## Отчёты

### Дашборд (`/dashboard/`)

Сводная страница с интерактивными графиками Chart.js:

- Итоговые карточки: численность, принято, уволено, коэффициент текучести
- Подкарточки: РСС, рабочие, разница за год
- Помесячная динамика движения персонала (Bar + Line)
- Текучесть по цехам (Vertical Bar с пороговыми линиями 10% и 20%)
- Причины увольнений (Doughnut)
- Причины увольнений по цехам (Stacked Bar)
- Численность по производствам (Stacked Bar: РСС + Рабочие)
- Таблица топ-5 цехов с наибольшей текучестью
- Фильтр по году

### Численность (`/reports/headcount/`)

Текущий срез численности по производствам и цехам с разбивкой на РСС и рабочих. Фильтры по производству и цеху. Экспорт в Excel.

### Движение персонала (`/reports/movement/`)

Годовой отчёт с четырьмя вкладками:

1. **Помесячная динамика** — принято, уволено, разница, численность на конец месяца
2. **Текучесть по цехам** — коэффициент текучести с цветовой индикацией
3. **Причины увольнений** — сводная таблица с визуальными полосками распределения
4. **Матрица цех × причина** — детальная разбивка увольнений

Фильтры по году, производству, цеху. Экспорт всех вкладок в Excel (многолистовый файл).

## Админ-панель

Доступна по адресу `/admin/` для пользователей с флагом `is_staff`. Позволяет:

- Управлять пользователями и их группами
- Редактировать справочники (производства, цеха, должности, категории, причины увольнений)
- Просматривать и редактировать данные сотрудников
- Назначать профили и ФИО пользователям системы
